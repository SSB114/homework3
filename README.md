# 鸢尾花分类与可视化实验 README

# 一、项目概述

本项目基于经典鸢尾花（Iris）数据集，完成多分类器对比、决策边界可视化及模型优化任务。核心目标是验证不同机器学习模型在小样本分类任务中的性能，通过 2D/3D 可视化直观呈现模型决策逻辑，并借助特征工程提升模型泛化能力。

项目分为 4 个核心任务，覆盖从基础分类到进阶优化的完整实验流程，同时配套详细的代码实现与可视化结果，适合机器学习入门实践与可视化方法学习。

# 二、环境配置

## 2.1 依赖库清单

运行本项目需安装以下 Python 库，建议使用 Python 3.8+ 版本：

- 核心计算：numpy（数组处理）、pandas（数据读取）

- 机器学习：scikit-learn（模型构建、数据预处理、评估）

- 可视化：matplotlib（2D/3D 绘图）、mpl_toolkits.mplot3d（3D 坐标轴支持）

- 文档编译（可选）：Overleaf 相关依赖（ctex、graphicx、subcaption 等，用于结果排版）

## 2.2 安装命令

通过 pip 快速安装依赖：

```bash

pip install numpy pandas scikit-learn matplotlib
```

# 四、核心任务与实现概述

## 4.1 Task1：多分类器2D决策边界对比

· 任务目标：构建 7 种经典分类器，对比其在鸢尾花数据集上的 2D 决策边界效果

· 核心实现：

- 配置分类器：逻辑回归、SVM、KNN、高斯朴素贝叶斯等 7 种模型

- 数据处理：选取 2 个特征，分层抽样划分训练/测试集

- 可视化：生成网格数据，绘制决策边界及 3 类样本的概率分布热力图

## 4.2 Task2：二分类3D决策超平面构建

· 任务目标：将多分类任务转换为二分类，构建 3D 特征空间的决策超平面

· 核心实现：

- 数据转换：将鸢尾花数据集转换为“山鸢尾 vs 其他”的二分类任务

- 特征标准化：对 3 个特征进行标准化处理

- 超平面推导：通过逻辑回归模型求解权重与偏置，推导 3D 决策超平面方程

- 3D 可视化：绘制 3D 散点图（样本分布）与决策超平面

## 4.3 Task3：3D概率分布可视化

· 任务目标：基于 Task2 的二分类模型，可视化 3D 空间内的概率分布

· 核心实现：

- 复用模型：直接使用 Task2 训练好的逻辑回归模型

- 3D 网格预测：生成 3D 网格数据，预测每个网格点的分类概率

- 多形式可视化：绘制概率曲面、等高线（Z 轴投影），直观呈现概率渐变规律

## 4.4 Task4：特征工程与模型优化

· 任务目标：通过特征工程提升模型泛化能力，实现增强型可视化

· 核心实现：

- 特征工程：构建 Pipeline 管道，完成多项式特征交互、标准化、特征选择

- 模型调优：优化逻辑回归模型参数，通过 5 折交叉验证评估性能

- 增强型可视化：基于优化后的模型，绘制更高精度的 3D 概率分布曲面

# 五、运行步骤

1. 环境配置：确保已安装 2.1 节所需的全部依赖库

2. 文件准备：将项目代码下载至本地，无需额外准备数据集（默认加载 sklearn 内置 Iris 数据集）

3. 运行任务：按任意顺序运行 task1.py ~ task4.py，命令示例：
        `python task1.py`

4. 查看结果：运行完成后，可视化图片会自动保存至 results 目录，可直接打开查看；同时控制台会输出模型准确率、泛化能力提升等关键指标

# 六、注意事项

- 运行代码前请确保 results 目录已创建，否则可能导致图片保存失败

- 若需使用自定义数据集，需将数据放置于 data 目录，并修改代码中数据集加载的路径与格式

- 3D 可视化结果可能因设备性能略有差异，可通过调整网格密度（np.meshgrid 中的点数）平衡清晰度与运行速度

- 若需在 Overleaf 中排版结果，需将 results 目录中的图片上传至 Overleaf 根目录，使用项目配套的 LaTeX 代码编译

# 七、常见问题解决

- 问题 1：运行时提示“模块找不到”—— 检查依赖库是否安装完整，建议重新执行 2.2 节的安装命令

- 问题 2：3D 绘图报错“no module named mpl_toolkits.mplot3d”—— 升级 matplotlib 至最新版本：`pip install --upgrade matplotlib`

- 问题 3：图片排版混乱（分页、抢占正文）—— 参考项目配套的 LaTeX 代码，清理重复的 geometry 包，调整图片尺寸与浮动参数（[ht]）
